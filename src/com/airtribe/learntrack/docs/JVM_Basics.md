# JVM Basics

## What is JDK, JRE, and JVM?

### JDK (Java Development Kit)
- **Full Form**: Java Development Kit
- **Purpose**: Complete development environment for Java
- **Contains**:
  - JRE (Java Runtime Environment)
  - Compiler (javac)
  - Debugger
  - Development tools (jar, javadoc, etc.)
- **When to use**: When you want to **develop** Java applications

### JRE (Java Runtime Environment)
- **Full Form**: Java Runtime Environment
- **Purpose**: Provides runtime environment to execute Java applications
- **Contains**:
  - JVM (Java Virtual Machine)
  - Core libraries (Java API)
  - Supporting files
- **When to use**: When you only want to **run** Java applications (not develop)

### JVM (Java Virtual Machine)
- **Full Form**: Java Virtual Machine
- **Purpose**: Executes Java bytecode
- **Function**: 
  - Loads bytecode
  - Verifies bytecode
  - Executes bytecode
  - Manages memory (heap, stack)
- **Platform-specific**: Each operating system has its own JVM implementation

### Relationship
```
JDK = JRE + Development Tools
JRE = JVM + Core Libraries
```

**Visual Representation**:

## What is Bytecode?

**Bytecode** is an intermediate representation of Java code that is:
- **Generated by**: Java compiler (javac)
- **Format**: `.class` files containing bytecode instructions
- **Platform-independent**: Same bytecode runs on any platform with JVM
- **Not machine code**: It's not directly executable by CPU
- **Executed by**: JVM interprets/compiles bytecode to machine code

### Compilation Process
```
Java Source Code (.java)
        ↓
    [javac compiler]
        ↓
   Bytecode (.class)
        ↓
      [JVM]
        ↓
   Machine Code
```

### Example
When you compile `HelloWorld.java`:
```bash
javac HelloWorld.java
```
This creates `HelloWorld.class` containing bytecode that JVM can execute.

## What Does "Write Once, Run Anywhere" Mean?

**"Write Once, Run Anywhere" (WORA)** is Java's core philosophy and one of its main advantages.

### Concept
- **Write Once**: You write your Java code once on any platform
- **Run Anywhere**: The same compiled bytecode can run on any platform that has a JVM installed

### How It Works
1. **Developer writes code** on any platform (Windows, Linux, Mac)
2. **Compile to bytecode** using `javac` - creates platform-independent `.class` files
3. **JVM executes bytecode** - each platform has its own JVM that converts bytecode to platform-specific machine code

### Example Scenario
```
Developer (Windows)          →  Compiles  →  HelloWorld.class (bytecode)
                                    ↓
User 1 (Linux)              →  Runs on Linux JVM  →  Executes successfully
User 2 (Mac)                →  Runs on Mac JVM    →  Executes successfully
User 3 (Windows)            →  Runs on Windows JVM →  Executes successfully
```

### Benefits
- **Portability**: No need to recompile for different platforms
- **Consistency**: Same behavior across platforms
- **Efficiency**: Write code once, deploy everywhere

### Trade-off
- **Performance**: Slight overhead due to JVM interpretation (though modern JVMs use Just-In-Time compilation to minimize this)
